About Weapons World 3D Application

1. Overview

My website is about weapons introduction. The website contains 3D models of three different weapons: 1911, Mossberg 500, and bow. And each weapon's 3D display has some interactive functions

2. 3D Models

I made three 3D weapon models in Blender. In order to make the animation better, I referred to the internal structure drawings of these weapons and made the different parts of the weapons separately. Each weapon has at least one swap animation and one firing animation.

3.Design

In addition to the content on canvas, I also learned more about html, css, js on W3Schools, MDN Web Docs, JavaScript.info

UI

I used a simple and clear web page layout. Each page has a follow-up navigation bar and each button has a touch animation effect and the edges are rounded to make it look more comfortable. In order to enhance the visual expression of the web page, libraries such as GSAP, ScrollReveal and Typed.js were added to enhance the visual effect.

3D Interaction

Each 3D model page provides the following features to the user:

Rotate manually or automatically.

Adjust background color

Adjust lighting.

Toggle wireframe.

Play specific animations and corresponding sound effects

4. Accessibility Considerations

Keyboard accessibility is supported for navigation and button interaction. Color contrast is sufficient for readability. Content layout adapts to different screen sizes. Labels and descriptions enhance usability for all users.

5. Test

Devices used: Windows personal computer (Chrome, Edge), Android phone (Chrome Mobile).

Browsers tested: Chrome, Edge.

Features tested: 3D Model, button interactivity, light control, background color change, animation triggering, page navigation.

Issues found:

There are errors in the paths of some buttons, resulting in some buttons being unable to enter the corresponding pages.

Some model animations are too slow.

Due to the size and position of the model, the camera position needs to be adjusted to better display it.

When playing animations, the sound and animation are out of sync.

Some code is repeated.

Final result: The webpage can run on major devices and browsers. All interactions and effects work and are smooth.

6. MVC

This project follows the Model-View-Controller (MVC) architecture principle:

Model: The 3D model paths, animation groups, and associated resources are stored and managed as arrays and objects in JavaScript, acting as the application's data layer.

View: The HTML and CSS components form the front-end structure that displays the 3D models, buttons, and user interface elements to users.

Controller: JavaScript functions handle all user interactions, trigger model loading, control animations, and dynamically modify the scene based on user inputs.

7. Deeper Understanding

In addition to the basic code, I also used GSAP, ScrollReveal, and Typed.js for some animation effects, which made the web page visually better and improved the user experience.

By referring to some models and pictures, and watching related videos(detailed content is on the Reference page), I made two relatively complex gun 3D models and made animations through keyframes.

Dynamic scene background color change and light control via JavaScript event handlers.
